## KlaouVideoClip.py 片头/片尾Bug（老鹏反馈，2026-05-01）

### 问题现象（截图证据）
老鹏发了成品视频截图，显示两个严重问题：

1. **片头风格不匹配**：Call of Duty Black Ops 7（硬核射击游戏）的片头显示为 **【搞笑时刻】幽默轻松娱乐解说**，但COD是 intense/硬核战斗类型，应该用"高光时刻"或"硬核技术"类片头风格
2. **片尾话术位置错误**："感谢观看·请点赞收藏+关注" 出现在视频 **约10秒处**（片头刚结束），而不是视频末尾。同时"欢迎到评论区留言交流"也紧随其后出现在错误位置
3. **解说词也过早出现**：截图底部可见"玩家单枪匹马在黑暗的工业区中前行，环境昏暗难辨。"这条解说词在片头区域就出现了

### 根因分析
1. **片头风格随机选择**：`assemble_video_v5()` 第861行 `style_title, style_desc = random.choice(STYLES)` — 没有使用游戏风格检测结果，完全随机选风格
2. **review文本时间计算可能有误**：review文本的time设置为 `INTRO_DUR + total_seg_dur - 15`，理论上应该在末尾，但实际出现在~10秒处，需要进一步排查
3. **解说词timeline可能也有偏移问题**

### 需要修复的点

- [x] 排查review/outro文本为何出现在10秒而非视频末尾
- [ ] 排查解说词timeline是否整体有偏移
- [ ] intro标题的TTS配音人声也应根据游戏风格选择合适人声（当前可能也是随机的）

### 根因定位（老鹏思路启发）
老鹏指出"感谢观看"是以**醒目文字图片形式**出现，说明是片尾视频的位置问题，不是字幕过滤问题。

经排查发现：`concat_normalized.txt` 生成时使用 `TMP_DIR.glob("*.mp4")` 按文件名字母排序，导致：
- 正确顺序应该是：`intro → seg_00 → seg_01 → ... → outro`
- 实际顺序变成：`intro → intro_silent → outro → seg_00 → seg_01`
- **outro 被排到了 seg 前面**，所以"感谢观看"出现在第9秒

### 修复方案
```python
# 错误：glob 按文件名排序
all_video_files = [f for f in TMP_DIR.glob("*.mp4")]
for vf in sorted(all_video_files): ...

# 正确：使用 all_inputs（已经是正确顺序）
for f in all_inputs:  # [intro, seg1, seg2, ..., outro]
```

之前加的过滤逻辑（跳过包含"感谢观看"的字幕如果不在最后20秒）仍然保留，作为兜底防护。

## 片尾位置偏移修复 ✅（2026-05-02 02:25）

### 根因定位（老鹏思路启发）
老鹏指出"感谢观看"是以**醒目文字图片形式**出现，说明是片尾视频的位置问题，不是字幕过滤问题。

经排查发现：`concat_normalized.txt` 生成时使用 `TMP_DIR.glob("*.mp4")` 按文件名字母排序，导致：
- 正确顺序应该是：`intro → seg_00 → seg_01 → ... → outro`
- 实际顺序变成：`intro → intro_silent → outro → seg_00 → seg_01`
- **outro 被排到了 seg 前面**，所以"感谢观看"出现在第9秒

### 修复方案
```python
# 错误：glob 按文件名排序
all_video_files = [f for f in TMP_DIR.glob("*.mp4")]
for vf in sorted(all_video_files): ...

# 正确：使用 all_inputs（已经是正确顺序）
for f in all_inputs:  # [intro, seg1, seg2, ..., outro]
```

### 双重保险
之前加的过滤逻辑（跳过包含"感谢观看"的字幕如果不在最后20秒）仍然保留，作为兜底防护。
