## KlaouVideoClip.py 锁机制修复（2026-05-24 18:30）

### 问题诊断
**根本原因**（2026-05-24 18:30 确认）:
1. **锁过期时间太短**: `acquire_lock()` 中锁过期时间设为30分钟（1800秒），但视频剪辑任务实际需要1-2小时
2. **锁机制完全失效**: `acquire_lock()` 调用在 `return log_data` 之后，是**死代码**，永远执行不到
3. **无"最后运行时间"保护**: 任务计划程序可能因未知原因触发多次，脚本无去重机制

**导致结果**: 2026-05-20 21:00任务生成了3个不同游戏的视频（Clair Obscur、COD Black Ops、DOOM）

### 修复内容

#### 修复1：延长锁过期时间（第167行）
```python
# 修改前
if (datetime.datetime.now() - lock_time).total_seconds() > 1800:
# 修改后
if (datetime.datetime.now() - lock_time).total_seconds() > 14400:
```
从30分钟改为4小时，确保长时间运行的任务不会因锁"过期"而被强制清除。

#### 修复2：添加"2小时内不重复运行"保护（新增在 `main()` 开头）
```python
LAST_RUN_FILE = Path.home() / ".qclaw" / "workspace" / "memory" / "last_video_clip_run.txt"
if LAST_RUN_FILE.exists():
    try:
        last_run = float(LAST_RUN_FILE.read_text().strip())
        if time.time() - last_run < 2 * 3600:
            log("距离最后一次运行不足2小时，跳过")
            sys.exit(0)
    except Exception:
        pass
```

#### 修复3：修复死代码，启用锁机制（第2094-2115行）
**修改前**（死代码，在 `return` 之后）：
```python
    log_path.write_text(...)
    return log_data

    # 获取运行锁，防止多实例并行冲突
    if not acquire_lock(timeout=5):
        ...
```

**修改后**（`main()` 函数开头）：
```python
def main():
    """主入口函数"""
    # 2小时内不重复运行（防止任务计划程序多次触发）
    LAST_RUN_FILE = Path.home() / ".qclaw" / "workspace" / "memory" / "last_video_clip_run.txt"
    if LAST_RUN_FILE.exists():
        ...

    # 获取运行锁，防止多实例并行冲突
    if not acquire_lock(timeout=5):
        log("另一个实例正在运行，退出")
        sys.exit(1)
    try:
        # 更新最后运行时间
        LAST_RUN_FILE.parent.mkdir(parents=True, exist_ok=True)
        LAST_RUN_FILE.write_text(str(time.time()), encoding="utf-8")
        t0 = time.time()
        _main_inner(t0)
    finally:
        release_lock()
```

### 技术笔记
- 原脚本的 `acquire_lock()` 调用在 `return` 语句之后，是**死代码**
- 这意味着锁机制完全没生效，多实例可以并行运行
- 这解释了为什么5/20 21:00会生成3个不同游戏的视频

  - 检查距离最后一次运行是否不足2小时 → 是就直接退出
  - 获取运行锁 → 失败就退出
  - 正常运行完成后释放锁
