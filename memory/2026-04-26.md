## 17:12 - KlaouVideoClip.py v5.1 Boss战关键帧保护

### 问题诊断
从老鹏手动剪辑的成品分析发现：两场Boss战（冰柜女性、机甲男人）都缺少**击倒瞬间的关键帧**。

### 改进方案（已实装）

#### 1. 新增 `detect_boss_fights()` 函数
- 用 `scdet` 检测高动作密度区间（10秒内>5次场景变化）
- 计算动作强度分数，识别持续8秒以上的激烈战斗

#### 2. 修改 `select_segments()` - 优先选择Boss战
- 先扫描所有文件的Boss战区间
- **优先截取包含Boss战的片段**：围绕Boss战前后各留3秒缓冲
- 确保Boss战片段长度在 SEG_MIN~SEG_MAX 范围内
- 普通片段避开已用的Boss战区间

#### 3. 修改 `trim_static_from_segment()` - 保护Boss战
- 新增 `boss_fights` 参数
- 静态裁剪时检查是否与Boss战区间重叠
- **重叠则跳过裁剪**，保护关键帧不被误删

#### 4. 修改 `generate_commentary()` - Boss战专用解说
- 新增 `boss_tpls` Boss战专用解说词
- Boss战片段的解说放在Boss战区间内（偏后，确保包含击倒瞬间）

- `high_motion_threshold=0.15` - 场景变化检测阈值

### 根本Bug确认（Step 7）
`KlaouVideoClip.py` 第~1160行附近，Step 7 BGM混音逻辑有**两处严重Bug**：

**Bug 1 - `chain_filters` 未初始化（NameError）**：
```python
if len(bgm_path) > 1:
    chain_filters.append(...)  # ❌ chain_filters从没被初始化！
```
→ 会触发 `UnboundLocalError`，导致整个 ffmpeg 命令不执行。

**Bug 2 - `bgm_all` 在单BGM时未定义**：
```python
if len(bgm_path) > 1:
    # bgm_all 在这里定义
    ...
else:
    bgm_chain = "[bgm0]"  # 但这里没有设置 bgm_all！
# 后面：delay_parts.append(f"...[bgm_all]...")  # ❌ NameError
```

当 len(bgm_path)==1 时：
1. `chain_filters` 未定义 → UnboundLocalError（即使后面没用）
2. `bgm_all` 未定义 → UnboundLocalError 在 delay_parts 构建时

### Bug修复（已执行）
在 `if len(bgm_path) > 1:` 块**之前**插入：
```python
chain_filters = []  # 修复Bug1：初始化
bgm_all = "[bgm0]"  # 修复Bug2：单BGM时默认[bgm0]
```

## 23:09 - 选片风格补充（老鹏反馈）

### 两种剪辑风格

#### 风格1：剧情/战斗向（优先）
- **核心原则**：不要出现角色失败、死亡画面 → 这些地方要剪掉
- 避免：被击杀、落命、倒地、过场死亡动画
- 目标：流畅连贯的观感，给观众正向能量

#### 风格2：幽默搞笑向（例外）
- **核心原则**：故意把出糗画面剪在一起
- 配幽默解说 + 搞笑BGM
- 观众反而喜欢看这种"翻车合集"

### 选片优先级（更新）
1. 完成度高（无失误/磕碰）
2. 动作流畅（摆荡/连招）
3. 无死亡/失败画面（剧情向）
4. 一镜到底 > 多镜切换

### 对脚本的启发
- 当前脚本是纯随机选片，无法区分这两种风格
- 未来改进方向：
  - 检测角色死亡/失败事件（画面突变、HP归零特效、音效）
  - 剧情向：默认过滤死亡帧
  - 搞笑向：主动收集失败帧
- 短期内：脚本保持随机，老鹏手动选片时按此标准筛选
