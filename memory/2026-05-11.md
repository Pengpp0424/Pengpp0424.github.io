### 正确工作流（已验证可行）
```
LoadImage → CLIPVisionLoader → CLIPVisionEncode
          → CLIPLoader(type=wan) → CLIPTextEncode ×2
          → UNETLoader(wan2.2_i2v_high_noise_14B_fp8_scaled)
          → WanImageToVideo(checkpoint+conditioning) ← 输出 positive/negative CONDITIONING + LATENT
          → KSampler(20步 euler) ← 真正的采样
          → VAEDecode → SaveAnimatedWEBP
```

### 模型配置
- UNETLoader: `wan2.2_i2v_high_noise_14B_fp8_scaled.safetensors` (14.3GB fp8)
- CLIPLoader: `umt5_xxl_fp8_e4m3fn_scaled.safetensors`
- CLIPVisionLoader: `clip_vision_h.safetensors`
- VAELoader: `wan_2.1_vae.safetensors`
- 分辨率: 832×480, 81帧, 20步

之前用 `Wan22ImageToVideoLatent` + `KSampler`，但 `VAEDecode` 输出像素图像而非 LATENT，导致噪点。

### 正确工作流
```
LoadImage → CLIPVisionEncode
          → CLIPLoader(type=wan) → CLIPTextEncode ×2
          → UNETLoader(wan2.2_i2v_high_noise_14B_fp8_scaled)
          → WanImageToVideo → KSampler(20步) → VAEDecode → SaveAnimatedWEBP
```

### 模型配置
- UNETLoader: `wan2.2_i2v_high_noise_14B_fp8_scaled.safetensors` (14.3GB fp8)
- CLIPLoader: `umt5_xxl_fp8_e4m3fn_scaled.safetensors`
- CLIPVisionLoader: `clip_vision_h.safetensors`
- VAELoader: `wan_2.1_vae.safetensors`
- 分辨率: 832×480, 81帧, 20步
