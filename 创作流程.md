# 视频创作工作流 v1.0

> 为老鹏定制的一站式游戏视频创作流程
> 
> 目标：高效产出高质量游戏视频，整合 AI 工具链

---

## 📊 流程图概览

```
素材采集 → AI辅助 → 视频剪辑 → 后期处理 → 发布分发 → 数据分析
   ↓           ↓          ↓          ↓          ↓          ↓
  OBS     ComfyUI    FFmpeg    调色/混音    Bilibili   播放量分析
 录屏     生图        预处理     字幕添加    抖音/快手    调整策略
```

---

## 1️⃣ 素材采集（Material Collection）

### 工具
- **OBS Studio**：游戏录屏（4K 60fps）
- **NVIDIA ShadowPlay**：即时回放（Last 5 mins）
- **Xbox Game Bar**：Win10/11 内置录屏

### 流程
1. **录制设置**：
   - 分辨率：4K (3840×2160) 或 2K (2560×1440)
   - 帧率：60fps
   - 编码器：NVENC (NVIDIA) 或 AMF (AMD)
   - 格式：MP4 (H.264)

2. **素材分类**：
   ```
   F:\适合剪辑\
   ├── Elden Ring\
   │   ├── 2026-05-24_Boss战_玛尔基特.mp4
   │   ├── 2026-05-24_探索_湖之路易.mp4
   │   └── ...
   ├── GoW Ragnarok\
   └── ...
   ```

3. **关键帧标记**：
   - 用 `ffprobe` 提取关键帧时间戳
   - 或用 Movavi 手动标记高光时刻

### 输出
- 原始素材（MP4）
- 关键帧时间戳列表（TXT）

## 2️⃣ AI 辅助制作（AI-Assisted Production）

### 工具链
| 工具 | 用途 | 路径 |
|------|------|------|
| **ComfyUI** | 生图（概念图/角色设计） | 本地 |
| **Edge TTS** | 配音（多角色声音） | `scripts/tts.py` |
| **smart_commentary.py** | 智能解说词生成 | `scripts/smart_commentary.py` |
| **LiblibAI API** | 文生图（低成本） | `scripts/liblib_api.py` |

### 流程
1. **概念图设计**（ComfyUI）：
   - 工作流：`FLF-720p.json`（81帧/镜头）
   - 输出：角色设计图、场景概念图
   - 保存路径：`F:\ComfyUI\output\`

2. **解说词生成**（smart_commentary.py）：
   - Tier 1：视觉分析 + AI 生成（gemma3:4b）
   - Tier 2：HUD 检测 + AI 生成
   - Tier 3：模板兜底
   - 输出：`scripts/output/commentary_*.txt`

3. **配音生成**（Edge TTS）：
   - 旁白：`YunjianNeural`（激情男声）
   - 角色对话：不同声音（YunxiaNeural / XiaoxiaoNeural）
   - 输出：`scripts/output/tts_*.mp3`

4. **图生视频**（LiblibAI）：
   - 目前仅支持网页端 LibTV（无 API）
   - 手动上传图片 → 选择模板 → 下载视频

### 输出
- 概念图（PNG/JPG）
- 解说词文本（TXT）
- 配音 MP3（~65 条/视频）
- AI 生成视频片段（MP4）

## 3️⃣ 视频剪辑（Video Editing）

### 工具
| 工具 | 用途 | 优势 |
|------|------|------|
| **FFmpeg** | 预处理（格式转换/剪辑） | 快速、命令行 |
| **Movavi** | 精细剪辑（特效/字幕/BGM） | 易用、模板丰富 |
| **moviepy** | Python 脚本剪辑（已弃用） | - |

### 流程
1. **预处理**（FFmpeg）：
   ```bash
   # 剪辑片段（起始时间 00:05:30，时长 10 秒）
   ffmpeg -i input.mp4 -ss 00:05:30 -t 10 -c copy output.mp4
   
   # 合并所有 TTS 配音为单条音轨（减少混音 bug）
   ffmpeg -f concat -i tts_list.txt -c copy tts_merged.mp3
   ```

2. **精细剪辑**（Movavi）：
   - 导入素材（视频 + 配音 + BGM）
   - 添加字幕（SRT 格式）
   - 添加特效（转场/滤镜）
   - 音频混音（原声 / 配音 / BGM 分轨处理）

3. **解说词同步**：
   - 用 `smart_commentary.py` 生成的解说词
   - 在 Movavi 中手动对齐时间轴

### 输出
- 剪辑工程文件（Movavi Project）
- 渲染输出视频（MP4，H.264，AAC）

## 4️⃣ 后期处理（Post-Processing）

### 工具
- **FFmpeg**：调色/音频混音/字幕烧录
- **Aegisub**：字幕制作（ASS 格式）
- **Audacity**：音频处理（降噪/均衡器）

### 流程
1. **调色**：
   ```bash
   # 增加对比度 + 饱和度
   ffmpeg -i input.mp4 -vf eq=contrast=1.2:saturation=1.3 output.mp4
   ```

2. **音频混音**（分轨处理）：
   - 轨道 1：游戏原声（volume=0.3）
   - 轨道 2：配音（volume=1.0）
   - 轨道 3：BGM（volume=0.5，淡入淡出）
   - 用 `ffmpeg filter_complex` 混音（先用 concat demuxer 合并 TTS）

3. **字幕烧录**：
   ```bash
   ffmpeg -i input.mp4 -vf subtitles=subtitle.ass output.mp4
   ```

4. **成品验证**（ffprobe）：
   ```bash
   ffprobe -v error -show_streams output.mp4
   # 确认 3 条音轨都存在
   ```

### 输出
- 最终成品视频（MP4，2K/4K，60fps，立体声）

## 5️⃣ 发布分发（Publishing & Distribution）

### 平台策略
| 平台 | 内容类型 | 发布策略 |
|------|----------|----------|
| **Bilibili** | 完整版（5-15 分钟） | 主阵地，B 站独家首发 |
| **抖音** | 短视频（1-3 分钟） | 高光片段，引导关注 B 站 |
| **快手** | 短视频（1-3 分钟） | 同抖音，不同剪辑风格 |
| **小红书** | 图文 + 短视频 | 教程向/幕后花絮 |
| **pengworks.com** | 作品展示 | 个人品牌官网 |

### 发布流程
1. **Bilibili**：
   - 标题：`【游戏名】标题（吸引点击）`
   - 标签：`#游戏 #解说 #老鹏`
   - 封面：高清截图 + 大字标题
   - 发布时间：周五 20:00（黄金时段）

2. **抖音/快手**：
   - 剪辑 1-3 分钟高光片段
   - 添加热门 BGM
   - 标题：`#游戏名 #操作 #高光`
   - 发布时间：每日 12:00 / 18:00

3. **pengworks.com**：
   - 更新 `data.json` 的 `works` 数组
   - 添加新作品（title / tag / bvid / description）
   - 提交 Git 并推送

## 6️⃣ 数据分析（Data Analysis）

### 指标
- **播放量**：24 小时 / 7 天 / 30 天
- **点赞率**：点赞数 / 播放量
- **评论互动**：回复评论，增加粘性
- **粉丝增长**：周增长 / 月增长

### 工具
- **Bilibili 创作中心**：数据分析
- **抖音创作者中心**：数据看板
- **Google Analytics**（pengworks.com）：访问量 / 用户行为

### 优化策略
1. **播放量低** → 优化标题 / 封面 / 前 5 秒内容
2. **点赞率低** → 内容质量提升 / 增加互动环节
3. **评论少** → 视频末尾引导评论（"你觉得呢？"）
4. **粉丝增长慢** → 增加更新频率 / 跨平台引流
